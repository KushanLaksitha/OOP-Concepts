Abstraction

අනවශ්‍ය විස්තර සඟවා සරල බව ඇති කිරීම (Hiding Complexity, Showing Simplicity)

Abstraction යනු කුමක්ද?

Abstraction යනු සංකීර්ණ ක්‍රියාවලීන්ගේ අභ්‍යන්තර විස්තර සඟවා, පරිශීලකයාට (user) අවශ්‍ය වන අත්‍යවශ්‍ය තොරතුරු පමණක් පෙන්වීමේ ක්‍රියාවලියයි. මෙහි අරමුණ වන්නේ සංකීර්ණත්වය අඩු කර, object එක භාවිතා කිරීම පහසු කිරීමයි.

Implementation Hiding: Object එකක් යම් කාර්යයක් "කරන්නේ කෙසේද" යන්න සඟවා, එය "කරන්නේ කුමක්ද" යන්න පමණක් පෙන්වීම.
Simplified Interface: පරිශීලකයාට object එක සමඟ ගනුදෙනු කිරීමට සරල interface එකක් (public methods) ලබා දීම.

සැබෑ ලෝක උදාහරණය: රූපවාහිනී දුරස්ථ පාලකය (TV Remote)

ඔබ TV remote එකක් භාවිතා කරන විට, ඔබ කරන්නේ බොත්තම් (buttons) එබීම පමණි. උදාහරණයක් ලෙස, power බොත්තම එබූ විට TV එක on හෝ off වේ, volume බොත්තම් වලින් ශබ්දය අඩු වැඩි වේ.

නමුත් එම බොත්තම එබූ විට ඇතුළත සිදුවන සංකීර්ණ ඉලෙක්ට්‍රොනික ක්‍රියාවලිය (signal යැවීම, circuit board එක ක්‍රියාත්මක වීම) ගැන ඔබ දැනුවත් නැත, දැන ගැනීමට අවශ්‍යත් නැත. Remote එකේ interface එක (බොත්තම්) මගින් ඒ සියලු සංකීර්ණත්වය ඔබෙන් සඟවා (abstract කර) ඇත.

C++ හි Abstraction ක්‍රියාත්මක කිරීම

C++ වල Abstraction ක්‍රම දෙකකින් ප්‍රධාන වශයෙන් ක්‍රියාත්මක කළ හැක:

  1. Access Specifiers (public, private): Encapsulation වලදී මෙන්, අභ්‍යන්තර දත්ත `private` ලෙස තබා public methods පමණක් ලබා දීමෙන් සරල Abstraction එකක් ලබා ගත හැක.
  2. Abstract Classes: වඩාත් දියුණු ක්‍රමයකි. මෙහිදී object එකක් සෑදිය නොහැකි, blueprint එකක් ලෙස පමණක් ක්‍රියා කරන class එකක් (Abstract Class) සාදනු ලබයි.

Abstract Class එකක අවම වශයෙන් එක් pure virtual function එකක්වත් තිබිය යුතුය. Pure virtual function යනු implementation එකක් නොමැති function එකකි. එය `= 0` ලෙස අවසන් වේ.

C++ 
#include <iostream>
using namespace std;
// Abstract Base Class
class Shape {
public:
// Pure virtual function - provides an interface
virtual void draw() = 0;
// Normal function
void describe() {
    cout << "This is a shape." << endl;
}
};
// Derived Class 1
class Circle : public Shape {
public:
// Provide implementation for the pure virtual function
void draw() {
cout << "Drawing a Circle: O" << endl;
}
};
// Derived Class 2
class Square : public Shape {
public:
// Provide implementation for the pure virtual function
void draw() {
cout << "Drawing a Square: []" << endl;
}
};
int main() {
// Shape myShape; // ERROR! Cannot create an object of an abstract class.
Circle myCircle;
Square mySquare;

myCircle.describe();
myCircle.draw(); // Calls the Circle's implementation

cout << "----------" << endl;

mySquare.describe();
mySquare.draw(); // Calls the Square's implementation

return 0;
}
Output:
This is a shape.
Drawing a Circle: O
----------
This is a shape.
Drawing a Square: []
මෙහිදී වැදගත් වන්නේ:
  • `Shape` class එකෙන් කියවෙන්නේ ඕනෑම shape එකක් `draw()` කළ යුතු බවයි, නමුත් එය "කරන්නේ කෙසේද" යන්න `Shape` class එක දන්නේ නැත.
  • `Circle` සහ `Square` classes එම `draw()` function එක තමන්ට ආවේණික ආකාරයට implement කරයි.
  • මෙය අනාගතයේදී `Triangle`, `Rectangle` වැනි තවත් shapes එකතු කිරීම පහසු කරයි. ඒ සෑම එකක්ම `draw()` function එක implement කළ යුතු බව `Shape` class එක සහතික කරයි.
