a) Ambiguity in multiple inheritance 03 marks
Ambiguity occurs when a derived class inherits from multiple base classes that have functions with the same name. C++ handles this by requiring explicit specification of which base class function to call. 
class A {
public:
    void display() { cout << "Class A"; }
};

class B {
public:
    void display() { cout << "Class B"; }
};

class C : public A, public B {
};

int main() {
    C obj;
    obj.A::display();  // Explicitly call A's display
    obj.B::display();  // Explicitly call B's display
}

c) Copy constructor explanation 03 marks
A copy constructor is a special constructor that creates a new object as a copy of an existing object. It takes a reference to an object of the same class as a parameter. 
class Box {
private:
    int width;
public:
    Box(int w) : width(w) {}
    
    // Copy constructor
    Box(const Box &obj) {
        width = obj.width;
        cout << "Copy constructor called" << endl;
    }
    
    void display() { cout << width << endl; }
};

int main() {
    Box b1(10);
    Box b2 = b1;  // Copy constructor called
}
d) Destructor explanation 03 marks
A destructor is a special member function that is automatically called when an object goes out of scope or is explicitly deleted. It has the same name as the class but is prefixed with a tilde (~). It's used to release resources allocated by the object. 
class Student {
private:
    string name;
public:
    Student(string n) : name(n) {
        cout << "Constructor called for " << name << endl;
    }
    
    ~Student() {  // Destructor
        cout << "Destructor called for " << name << endl;
    }
};

int main() {
    Student s("John");
    return 0;
}  // Destructor automatically called here

b) Why abstraction is essential 03 marks
Abstraction is essential for real-world applications because:
  • Complexity Management: Hides complex implementation details and shows only essential features
  • Code Reusability: Provides a clear interface that can be reused across different parts of the application
  • Maintainability: Changes to implementation don't affect code that uses the abstraction
  • Security: Restricts access to internal data and exposes only necessary functionality
  • Modularity: Allows developers to work on different components independently
c) Overloading vs Overriding 03 marks
Function Overloading:
  • Multiple functions with the same name but different parameters
  • Occurs in the same class
  • Compile-time polymorphism
  • Example: void print(int), void print(string)
Function Overriding:
  • Redefining a base class function in derived class
  • Occurs in inheritance hierarchy
  • Runtime polymorphism
  • Same function signature as base class
d) Protected inheritance effect 02 marks
When protected inheritance is used:
  • Public members of base class become protected in derived class
  • Protected members of base class remain protected in derived class
  • Private members of base class remain inaccessible in derived class

e) const keyword in functions 03 marks
The const keyword after a member function declaration indicates that the function is a const member function. This means: 
int getArea() const {
    return length * width;
}
  • Cannot modify member variables: The function promises not to change any member variables of the object
  • Can be called on const objects: Const objects can only call const member functions
  • Better code safety: Compiler enforces that the function doesn't accidentally modify the object's state
  • Example: Getter functions should typically be const since they only read data without modifying it
f) Observer design pattern 03 marks
The Observer design pattern is a behavioral design pattern where:
  • Purpose: Defines a one-to-many dependency between objects so that when one object (subject) changes state, all its dependents (observers) are notified automatically
  • Usage:
    • Event handling systems (GUI applications)
    • Model-View-Controller (MVC) architecture
    • Notification systems
    • Real-time data monitoring
  • Key Components:
    • Subject: Maintains list of observers and notifies them of state changes
    • Observer: Receives updates from the subject
  • Example: A weather station (subject) that notifies multiple display devices (observers) when temperature changes
