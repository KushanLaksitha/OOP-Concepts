Lesson 1: The Basic Class

What is a Class Diagram?

A Class Diagram is a blueprint used to describe the structure of a system. It shows the system's classes, their attributes, methods, and the relationships among them. The most basic component is the class itself.

Example: A 'Car' Class
UML Class Diagram:
Car
C++ Code:
class Car {
}; // An empty class definition

Lesson 2: Adding Attributes

What are Attributes?

Attributes are the data or properties that a class holds. In C++, these are the member variables. In a UML diagram, they are listed in the second compartment. The minus sign (-) indicates they are `private`.

Example: 'Car' with Attributes
UML Class Diagram:
Car
  • brand: string
  • year: int
C++ Code:
#include <string>
using namespace std;
class Car {
private:
string brand;
int year;
};

Lesson 3: Adding Methods

What are Methods?

Methods (or Operations) are the actions or functions that a class can perform. In C++, these are member functions. They are listed in the third compartment of the class diagram. The plus sign (+) indicates they are `public`.

Example: 'Car' with Methods
UML Class Diagram:
Car
  • brand: string
  • year: int
  • setBrand(b: string): void
  • getBrand(): string
C++ Code:
class Car {
private:
string brand;
int year;
public:
void setBrand(string b) {
brand = b;
}
string getBrand() {
return brand;
}
};

Lesson 4: Relationships

Inheritance

Inheritance is an "is-a" relationship. One class (the child) inherits attributes and methods from another (the parent). It is shown with a solid line and a hollow arrowhead pointing to the parent class.

Example: 'ElectricCar' is a 'Car'
UML Class Diagram:
Car
  • brand: string
  • getBrand(): string
ElectricCar
  • batteryLife: int
  • charge(): void
C++ Code:
// Parent Class
class Car { ... };
// Child Class inherits from Car
class ElectricCar : public Car {
private:
int batteryLife;
public:
void charge() {
// charging logic...
}
};

Lesson 5: Complete Example

Student and Course System

Let's combine everything we've learned to model a simple university system. A `Person` is a base class. A `Student` is-a `Person`. A `Student` can also enroll in a `Course` (this is an Association relationship, represented as a member variable).

Diagram & Code
UML Class Diagram:
Person
  • name: string
  • getName(): string
Student
  • studentID: int
  • enrolledCourse: Course*
  • enroll(c: Course*): void
C++ Code:
// Forward declaration for Course
class Course;
class Person { ... };
class Student : public Person {
private:
int studentID;
Course* enrolledCourse;
public:
void enroll(Course* c) {
enrolledCourse = c;
}
};
